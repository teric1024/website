    Goal of These Walkthroughs

    The primary objective of these walkthroughs is to sharpen pentesting skills. It is crucial to obtain root access as swiftly as possible, akin to executing a Real-Time Attack. The walkthroughs serve as simplified penetration testing reports. It is essential to ensure that each walkthrough includes:

    1. Path to Grant Root Access
    2. This section should detail the specific steps required to obtain root access, emphasizing clarity and accuracy.
    3. Vulnerabilities and Risks Classified by Severity
    4. Vulnerabilities should be identified and categorized based on their severity level.
    5. Evidence of Each Vulnerability and Risk
    6. Evidence should be provided to convincingly demonstrate that the specific vulnerability exists.
